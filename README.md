# Open CYD Player

A fully open-source MJPEG video player with synchronized MP3 audio for the **ESP32-2432S028R** — the 2.8" resistive-touch "Cheap Yellow Display" (CYD). Clean-room implementation, MIT licensed: firmware, converter, and file format are all open.

## Features

- MJPEG playback at 320x240 up to ~30 fps with frame-drop catch-up
- Synchronized MP3 audio via external PCM5102A I2S DAC (or the onboard speaker amp)
- Touch controls: tap to play/pause, ±1 minute seek buttons, tap-to-seek progress bar
- Edge gestures: drag the left edge for brightness, right edge for volume
- SD-card file browser with paging
- `.idx` frame index for instant seeking (open format, generated by `tools/convert.py`)
- Plays index-less `.mjpeg` files too (stream parsing, no seek)

## Hardware

| Part | Notes |
|---|---|
| ESP32-2432S028R (CYD 2.8" resistive) | ILI9341 display, XPT2046 touch |
| MicroSD card, FAT32 | faster cards = higher sustainable fps |
| PCM5102A I2S DAC (optional, recommended) | BCK→GPIO4, LRCK→GPIO22, DIN→GPIO27 |

On the PCM5102A board, bridge the SCK/FLT/DMP/FMT/BYP pads to GND so it generates its own system clock. No DAC? Set `USE_EXTERNAL_I2S_DAC 0` in `src/config.h` to use the onboard mono speaker output (GPIO26) — expect tinny audio.

## Build & flash

1. Install [PlatformIO](https://platformio.org) (VS Code extension or CLI).
2. `pio run -t upload` with the CYD connected over USB.
3. All display pins are configured via build flags in `platformio.ini` — no TFT_eSPI `User_Setup.h` editing needed.

## Convert videos

```bash
python tools/convert.py movie.mp4 --fps 24 --quality 8
```

Requires ffmpeg on PATH. This produces `movie.mjpeg`, `movie.mp3`, and `movie.idx`. Copy all three into a `/videos` folder on the SD card. Audio and index files must share the video's basename.

Tips: 24 fps is a safe default; try 30 with a fast SD card. Quality 6–10 balances size vs. clarity at 320x240.

## Touch calibration

Default calibration constants live in `src/config.h` (`TOUCH_X_MIN/MAX`, `TOUCH_Y_MIN/MAX`). If taps land offset, print raw values from the XPT2046 at the screen corners and adjust — an on-screen calibration wizard is a good first contribution.

## SD card layout

```
/videos/
  movie.mjpeg
  movie.mp3
  movie.idx
```

## .idx file format ("CYD1", little-endian)

| Offset | Size | Field |
|---|---|---|
| 0 | 4 | magic `CYD1` |
| 4 | 4 | fps × 1000 (uint32) |
| 8 | 4 | frame count (uint32) |
| 12 | 4×(n+1) | byte offset of each frame, plus file size sentinel |
